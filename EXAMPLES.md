## 日期格式示例

系统支持以下日期格式：
- `2025-01` / `2025/01` / `2025.01`
- `2025-1` / `2025/1` / `2025.1`
- `2025年1月`
- `202501`

## 测试流程

1. 准备两个测试文件（文件A和文件B）
2. 访问应用主页
3. 上传文件A（数据源文件）
4. 上传文件B（数据缺失文件）
5. 点击"开始处理"
6. 等待处理完成
7. 点击"下载结果"
8. 检查结果文件是否正确填充

## 预期结果

处理完成后，文件B的空白单元格应被填充为：
- 从文件A中找到的匹配数据
- 经过单位换算（如需要）
- 符合百分比格式（如需要）
- 字段名称通过同义词匹配

## 常见问题

### Q: 为什么有些单元格没有被填充？
A: 可能原因：
- 字段名称不匹配，请检查同义词是否在支持列表中
- 日期格式不支持，请使用标准格式
- 数据源文件中确实没有对应数据

### Q: 单位换算不正确怎么办？
A: 确保单位标注格式正确：
- 正确：`单位：亿元`、`单位：万元 %`
- 错误：`单位=亿元`、`亿元`（缺少"单位："前缀）

### Q: 百分比处理不准确？
A: 检查：
- 表格外是否正确标注了 `%`
- 数据源中的百分比是小数形式（0.45）还是百分比形式（45）

### Q: Word 文件解析失败？
A: 确保：
- Word 文件格式为 `.docx`（不支持 `.doc`）
- 表格使用标准 Word 表格格式
- 或者使用制表符分隔的文本

```typescript
// src/lib/data-utils.ts
const SYNONYMS: Record<string, string[]> = {
  '总资产': ['资产总额', '资产合计', '总资', '资产总计', '总资产额', '你的自定义同义词'],
  // 添加更多...
};
```
