## 如何使用

### 步骤 1: 准备文件

**文件A（数据源文件）示例：**
```
时间    总资产    净利润    营业收入
2025-01  1000.5    150.2     500.3
2025-02  1050.8    160.5     520.4
2025-03  1100.2    170.8     540.5
```

**文件B（数据缺失文件）示例：**
```
单位：万元

时间    资产总额    净利    营收
2025-01  
2025-02  
2025-03  
```

### 步骤 2: 上传文件

1. 点击"文件A（数据源文件）"的上传按钮
2. 选择包含完整数据的文件
3. 点击"文件B（数据缺失文件）"的上传按钮
4. 选择需要填充数据的文件

### 步骤 3: 处理数据

1. 点击"开始处理"按钮
2. 等待处理完成（会显示"数据处理完成！共填充 X 个单元格"）
3. 如果处理失败，会显示具体的错误信息

### 步骤 4: 下载结果

1. 处理成功后，"下载结果"按钮会从"等待处理"变为"下载结果"
2. 点击"下载结果"按钮
3. 下载填充后的 Excel 文件

## 常见问题

### Q1: "下载结果"按钮一直是灰色的？

**原因**: 处理失败了，没有成功生成文件

**解决方法**:
1. 打开浏览器控制台（F12），查看错误信息
2. 确保上传的文件格式正确（.xlsx, .xls, .docx）
3. 确保文件中包含表格数据
4. 检查服务器日志：`tail -n 50 /app/work/logs/bypass/app.log`

### Q2: 处理成功但填充了 0 个单元格？

**可能原因**:
1. 字段名称不匹配
2. 日期格式不支持
3. 数据源中没有匹配数据

**解决方法**:
1. 检查字段名称是否支持同义词匹配
2. 使用标准日期格式（2025-01, 2025/01, 2025年1月）
3. 确保数据源文件中确实有对应时间的数据

### Q3: 处理报错 "cannot save file"？

**原因**: 文件名包含特殊字符

**解决方法**: 已修复，现在会自动清理文件名中的特殊字符

### Q4: Word 文件解析失败？

**解决方法**:
1. 确保文件是 .docx 格式（不是 .doc）
2. 表格使用标准 Word 表格格式
3. 或者使用制表符分隔的文本

## 支持的同义词

| 标准字段 | 可匹配的字段 |
|---------|-------------|
| 总资产 | 资产总额、资产合计、总资 |
| 净利润 | 净利、净利润额、纯利润 |
| 营业收入 | 营收、收入、销售收入 |
| 营业成本 | 成本、营业成本额 |

## 支持的日期格式

- `2025-01` / `2025/01` / `2025.01`
- `2025-1` / `2025/1` / `2025.1`
- `2025年1月`
- `202501`

## 支持的单位

- 亿元
- 万元
- 元
- %（百分比）

## 文件要求

### Excel 文件
- 第一行应该是表头（字段名）
- 第一列应该是时间
- 数据单元格可以是数值或空白

### Word 文件
- 使用标准 Word 表格
- 表格外可以添加单位标注
- 例如：`单位：万元`

### 单位标注格式

**正确格式**:
- `单位：亿元`
- `单位：万元`
- `单位：万元 %`

**错误格式**:
- `单位=亿元`（使用等号）
- `亿元`（缺少前缀）
