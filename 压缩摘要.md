# 压缩摘要

## 用户需求与目标
- 原始目标: 创建智能表格数据填充工具，支持多种模式（常规、授信、PDF）进行数据匹配和填充。
- 当前目标: 
  - ✅ 已完成：实现PDF模式：识别扫描版PDF表格，提取机构名称和授信品种及金额，匹配并填充到Excel文件。
  - ✅ 已完成：修复PDF转图片功能，确保PDF模式正常工作。

## 项目概览
- 概述: 智能表格数据填充工具，支持上传数据源文件和缺失文件，自动识别表格结构、匹配字段、填充数据并导出。
- 技术栈:
  - Next.js 16 (App Router)
  - React 19
  - TypeScript 5
  - shadcn/ui
  - Tailwind CSS 4
  - xlsx (常规模式Excel处理)
  - exceljs (授信模式/PDF模式Excel处理，保留格式)
  - **pdftoppm (poppler-utils，PDF转图片)**
  - coze-coding-dev-sdk (LLM调用)

## 关键决策
- 使用 Vision LLM (`doubao-seed-1-6-vision-250815`) 进行PDF OCR识别
- PDF处理流程：PDF → pdftoppm转PNG → Vision LLM识别 → 提取数据
- **使用系统工具pdftoppm替代Node.js库**（pdf2pic/pdfjs-dist/canvas均因原生模块依赖问题无法使用）
- 三种处理模式：常规模式、授信模式、PDF模式
- **公式保留规则**：
  - 单体表D列：设置SUM公式（如 `=SUM(E4:AC4)`）
  - 集团表E列：授信品种金额之和（值）
  - 集团表C列：加法公式（如 `=E4+E5`）
  - 只清理共享公式，保留普通公式
- **文件名规则**：
  - 输出文件以日期前缀开头（YYMMDD格式）
  - 如果原文件已以YYMMDD开头，则替换为当前日期
  - 示例：`250315测试.xlsx` → `260319测试.xlsx`

## 问题或错误及解决方案
- **问题**: pdf2pic依赖GraphicsMagick/ImageMagick，沙箱环境未安装
  - **解决方案**: 尝试pdf-to-img，但有ESM兼容性问题
- **问题**: pdf-to-img与Next.js存在ESM兼容性问题
  - **解决方案**: 尝试pdfjs-dist + canvas
- **问题**: pdfjs-dist动态导入仍报错"Object.defineProperty called on non-object"
  - **解决方案**: canvas原生模块无法编译安装
- **问题**: canvas原生模块缺少系统依赖(pkg-config, pixman等)
  - **解决方案**: 安装poppler-utils，使用pdftoppm命令行工具转换PDF ✅
- **问题**: 导入错误(mkdirSync, rmSync, existsSync从child_process导入)
  - **解决方案**: 修正为从fs模块导入 ✅
- **问题**: 文件内容未正确更新，旧代码残留
  - **解决方案**: 重新写入正确的pdf-matcher.ts文件内容 ✅
- **问题**: 共享公式错误"Shared Formula master must exist above and or left of clone for cell E5"
  - **根本原因**: ExcelJS在writeBuffer时会验证共享公式的完整性，修改单元格值后可能触发验证错误
  - **解决方案**: 
    1. **清理所有公式** - 在数据处理完成后，将所有公式转换为值
    2. **手动计算汇总** - 自己计算汇总值填入单体表D列、集团表E列
    ✅ **已修复**
- **问题**: PDF转换失败"PDF文件转换失败，请确保PDF文件格式正确"
  - **根本原因**: poppler-utils未安装，pdftoppm命令不存在
  - **解决方案**: 执行`apt-get install -y poppler-utils`安装 ✅
- **问题**: 汇总公式丢失，填充数据后汇总数没有自动重新计算
  - **根本原因**: 无法保留Excel公式，共享公式验证会报错
  - **解决方案**: 
    1. **手动计算汇总值** - 自己计算E列和C列的值
    2. **单体表D列** - E列开始的授信品种金额之和
    3. **集团表E列** - F列开始的授信品种金额之和
    4. **集团表C列** - 合并单元格对应的E列求和
    5. **保留C列公式** - 清理时跳过C列，保留原有公式
    ✅ **已修复**
